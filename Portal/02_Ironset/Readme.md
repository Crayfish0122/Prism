# Head Fitness Coach — 健身训练与日程管理系统

个人体态重组（Body Recomposition）项目，由 Claude 担任 Chief of Staff，统筹管理力量训练、营养追踪、睡眠恢复与工作日程。

---

## 项目背景

基线数据（2026年初）：体重约73kg，体脂率约15–18%。参考体态照片拍摄于2026年4月，主要待改善区域为胸部厚度、肩部正面轮廓，以及下腹和下胸部的脂肪覆盖。

---

## 核心架构

### Claude 的角色

Claude 作为 Chief of Staff，每次对话启动时同时读取三个 Google Calendar（训练、睡眠、工作），自动生成简报：

1. **恢复状态** — 近5天平均睡眠时长 + 累计债务变化 + 一句结论
2. **营养概况** — 近5天平均摄入与达标程度
3. **今日训练建议** — 基于轮转计划和恢复状态

### 训练计划

四日轮转分化（Push / Pull A / Pull B / 腿），训练周定义为周日至周六。

| 日 | 内容 |
|---|---|
| Push | 悍马上斜推胸 · 蝴蝶机 · 悍马推肩 · 哑铃侧平举 |
| Pull A（背阔为主） | 器械T杠划船-宽距 · 器械高位下拉 · 直臂下压 · 悍马反手高位下拉 |
| Pull B（后肩/中背为主） | 反向蝴蝶机 · 面拉 · 蝴蝶机反向飞鸟 · 器械T杠划船-宽距 |
| 腿 | 坐姿腿屈伸 · 坐姿腿弯举 · 悍马臀推 · 悍马髋外展 · 坐姿髋内收 |

权威训练计划来源：[Google Doc](https://docs.google.com/document/d/1O0l4vx91yJD0dsWzbwXWkpCEauRBbUa-nMxPiZYkiN0/edit)

### 营养基线（2026年3月均值）

~2167 kcal/日 · ~234g碳水 · ~160g蛋白质 · ~66g脂肪。蛋白质≥130g达标率100%，热量控制在2000–2460 kcal区间内，无超标。

数据来源：`HealthData.xlsx`（含 Summarize / sleep_daily_raw / workout_daily_raw / nutrition_total 四个sheet）

## 日历配置

| 日历 | 用途 |
|---|---|
| 训练日历 | 记录每日训练内容、营养CPF数据、训练反馈 |
| 睡眠日历 | 睡眠时长与债务追踪 |
| 工作日历 | 会议与工作安排（仅工作日读取） |

时区：`Asia/Tokyo`。查询时始终使用 `condenseEventDetails=false` 以获取 description 中的完整数据。

---

## 训练反馈协议

训练完成后报告实际执行情况，Claude 将反馈写回日历 event 的 description，格式如下：

```
{原始计划内容}
===
训练日 |
动作名 | 实际重量×组次 | →下次建议
疲劳指数 | 1-10
分析 | 睡眠状态+营养状态 | 原因判断 | 建议
```

疲劳指数≥7时触发 deload 考量。变更训练日为休息日时，title 格式为 `休息日（XXX顺延）YYYY-MM-DD`。

---

## 工作日程管理

工作日读取工作日历后，根据上班时间扣除已有会议，评估可用工作时间：碎片时间建议处理邮件/审阅，整块时间建议深度工作。周末及节假日不读取工作日历。

---

## 规则摘要

- 回复简洁，结论先行，不罗列原始数据
- 训练/营养/恢复建议须有运动科学文献精度
- 日历更新仅在明确要求时执行
- 睡眠追踪使用债务增量系统（正值=累积赤字，负值=恢复盈余）
