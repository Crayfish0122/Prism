# Head Fitness Coach — 健身训练管理系统

个人体态重组(Body Recomposition)项目,由 Claude 担任 Head Fitness Coach,主线统筹力量训练,睡眠/营养/工作仅作为背景变量用于解释训练表现或安排训练时间。

---

## 项目背景

基线数据(2026年初):体重约73kg,体脂率约15–18%。参考体态照片拍摄于2026年4月,主要待改善区域为胸部厚度、肩部正面轮廓,以及下腹和下胸部的脂肪覆盖。

---

## 核心架构

Claude 作为 Head Fitness Coach,通过四个任务覆盖训练管理全流程。具体执行规则、字段定义、决策阈值、写回格式全部以 `Prompt.txt` 为准,README 不复述 Prompt 内容。

### 四个核心任务

任务一 开启健身训练。提示词:干活啦/起床/起床干活啦。读取近5天睡眠/营养/训练/生活日历,工作日加读工作日历,输出恢复状态、营养概况、今日训练建议、可用工作时间评估、未来计划评估。

任务二 健身反馈输出。提示词:反馈/输出。训练后生成反馈草稿,包含结构化记录层(写入 event)与教练短评层(单动作分析),经确认后追加写回当日训练 event 的 description。

任务三 变更健身计划。提示词:变更。生成日历变更草稿(改休息日时同时改 title,仅调整动作时只改 description),经确认后写回。

任务四 围度记录。提示词:记录。双周一次,周日空腹,生成新建 event 草稿(围度数据)经确认后创建。

任务二/三/四 默认采用两阶段模式:先出草稿,经明确确认后才执行日历写回。

## 固定数据源

| 数据源 | 用途 |
|---|---|
| 肉体改造(训练日历) | 每日训练计划与反馈写回 |
| 灵魂返厂(睡眠日历) | 睡眠时长与债务追踪 |
| 烟火知味(营养日历) | 每日 CPF 摄入与达标记录 |
| 人间烟火(生活日历) | 生活事件背景变量 |
| 搬砖手册(工作日历) | 工作日会议与可用时间(仅工作日读取) |
| Exercise Reference Table | 动作配置主表(角色/目标组次/工作重量/风险标签/技术关注点/进步判定/替换条件/替代动作) |
| HealthData/Summarize | 未来5天计划主表(只读 I、J 列) |

权威训练计划来源:[Google Doc](https://docs.google.com/document/d/1O0l4vx91yJD0dsWzbwXWkpCEauRBbUa-nMxPiZYkiN0/edit)

时区:`Asia/Tokyo`。

---

## 读取优先级

当天训练日与动作顺序以训练日历 event 为准。动作管理、当前工作重量、风险标签、技术关注点以 Exercise Reference Table 为准。某动作最近练成什么样以训练日历近期记录为准。HealthData/Summarize 仅用于未来5天计划参考,不覆盖当天训练 event。
