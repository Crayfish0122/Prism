### Revision History

### [v0.1.0] - 2026-04-16

新增
新增 任务一 开启健身训练,定义读取范围(灵魂返厂/肉体改造/烟火知味/人间烟火 过去5天,工作日加搬砖手册)、读取字段、内部提取与缓存、分析目标(恢复状态/营养概况/今日训练建议/工作日分支/未来计划评估)五段流程
新增 任务二 健身反馈输出,定义训练后反馈草稿格式,包含训练前状态、RPE分组、完成情况、下次方案、动作质量、风险标签、下次重量、全局结论与单动作分析
新增 任务三 变更健身计划,定义训练日改休息日的 title 格式 与仅调整动作时只更新 description 的分支
新增 任务四 围度记录,定义双周一次周日空腹的围度 event 创建规则
新增 两阶段模式,任务二/三/四 默认先生成草稿,经明确确认后才写回日历
新增 缓存规则,任务一执行时缓存当天 event_id 与全部上下文,任务二/三 默认复用当日缓存
新增 固定数据源 与 读取优先级,明确训练日历、Exercise Reference Table、HealthData/Summarize 三者冲突时的取舍
新增 决策规则,要求输出训练建议前必须校验连续训练天数、昨日疲劳指数、近5天 sleep debt 变化、deep sleep 异常
新增 动作决策规则,定义加重/维持/减重的 RPE 阈值与连续2次风险触发的替换提示
新增 动作底表使用要求,引用当前状态、风险标签、技术关注点、进步判定方式、替换触发条件、替代动作六个字段
新增 工作日分支,读取搬砖手册后询问上班时间,扣除会议后按碎片/整块时间给建议
新增 未来计划评估,基于 HealthData/Summarize 未来5天判断如期/激进/松弛
新增 时间窗口定义,"近5天"=D-4~D,"过去5天"=D-5~D-1

修改 日历命名,训练/睡眠/工作/营养 改为 肉体改造/灵魂返厂/搬砖手册/烟火知味,新增 人间烟火
