# 变更履历 · 烟火知味

> 记录 Prompt 各版本的核心变更与优化方向。

---

## v1.1.0（Prompt 结构优化版）

**变更方向**：提升指令清晰度与模型执行稳定性

### 变更内容

**新增「核心红线」区块**：将最高优先级约束（禁止编造、不得串任务、必须等待确认等）从任务描述中抽离，统一置于 Prompt 开头，确保模型在任何任务路径下都优先加载这些约束。

**任务二输出规则细化**：
- 明确区分「首次基线加载」和「后续报餐」两个状态，防止模型重复询问「今天记过吗」
- 新增简洁模式说明：后续报餐省略基线明细，仅在用户主动要求时才展开
- `warned_flags` 去重机制补充说明，避免同类警告在 session 内反复触发

**任务三失败分支完善**：
- 区分「主表写失败」和「主表成功但日志失败」两种不同的失败结果，并给出不同的处理策略和用户提示
- 明确 `retained_request_id` 在主表写失败时应保留、在成功后应清除

**任务一输出格式锁定**：在定调流程末尾明确「固定输出仅三块，不得增删」，防止模型自由发挥额外内容。

**全局约定补充**：新增 session 缓存变量完整列表（`warned_flags`、`retained_request_id` 等），以及「任何任务涉及 session 新值/覆盖/清理时必须显式写回」的强制要求，堵住只在回复中体现但不更新缓存的漏洞。

## 已知限制与后续优化方向

**限制**

- 时区硬编码为 `Asia/Tokyo`，跨时区使用需手动修改 Prompt
- `User_Vocab` 词典在 session 首次进入任务二时只读一次，当次 session 内无法感知词典新增内容
- 日历读取失败会直接终止反馈任务，无降级策略

**后续优化方向**

- 考虑引入「快速记录模式」，减少首次基线询问的打断感
- 评估是否需要支持跨日汇总对比（周报 / 月报）
- 探索将 `User_Vocab` 读取移至 session 初始化阶段，提升词典命中一致性
- 研究训练内容与营养目标的更细粒度关联（如力量日 vs 有氧日的碳水差异化策略）

## v1.0.0（初始版本）

**发布时间**：2025 年（初版）

### 功能覆盖

- 实现四个核心任务：GetDayType / CreateMealDraft / AppendFoodRows / DailyNutritionFeedback
- 实现第五任务：AppendBestNewVocab（个人词典新增）
- 定调流程支持训练日 / 休息日双桶分离计算
- 支持放纵日声明与幂等写入校验（`request_id` 机制）
- 支持图片、文字、包装识别多种摄入输入方式
- Session 缓存机制：`today_target`、`today_baseline`、`today_draft` 等核心变量全部纳入管理

### 核心设计决策

**幂等写入**：引入 `append_log` 日志表，每次写入前先查重 `request_id`，从根本上防止重复记录。

**双桶负债计算**：训练日与休息日分开建桶，各自计算过去 7 天平均日负债，动态调整当日目标，而非使用固定目标值。

**强触发设计**：写入操作要求明确触发词（写入 / 保存 / 提交 / 记录到表格），避免误操作写入；反馈也仅在明确要求时输出，防止打扰日常记录流程。

**数据诚实原则**：拿不准的条目强制标「待核实」；写入失败不假装成功；日历读取失败不允许继续输出分析结果。
