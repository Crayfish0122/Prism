# 饮食记录与营养分析助手

这是一个基于大语言模型（ChatGPT / Claude）的个人饮食数据记录与营养分析 prompt，配合 Google Sheets 和 Google Calendar 使用。它帮你做四件事：每天根据训练/休息日动态计算营养目标、用自然语言记录每一餐、把草稿幂等地写入表格、并在你需要时给出当日饮食反馈。所有数据都存在你自己的 Google Drive 里，助手只负责读写和计算，不编造数据、不简化流程。

## 使用前你需要准备什么

首先是四张 Google Sheets。主表叫 `meal_daily_raw`，固定 16 列，按顺序是：date, type, kcal, carb, protein, fat, unit, food, brand, note, source, updated, scene, intake_time, inserted_at, request_id。另外三张分别是 `cheat_days`（放纵日记录）、`append_log`（写入日志，用于幂等校验）和 `User_Vocab`（你的个人食物词典）。

其次是一个 Google Calendar，建一个叫"烟火知味"的即可。训练日在对应日期的事件 title 或 description 里写上训练内容，休息日不用管。

最后，把 `spreadsheet_id` 和 `Calendar ID` 写进模型的长期记忆（ChatGPT Memory 或对应平台的等价功能），助手每次从那里取。时区硬编码为 `Asia/Tokyo`，在别的时区需要自己改 prompt。

## 四个核心任务

**任务一：定调（GetDayType）**。每天起床说一句"起床了""早上好"或"干活"，助手会读今日日历判断训练日还是休息日，再往回扒过去 7 天的实际摄入，算出训练日桶和休息日桶各自的平均日负债，动态调整今日的 Kcal/P/C/F 目标。训练日基础目标 2460 / P160 / C320 / F60，休息日 2150 / P160 / C220 / F70，会在这基础上按硬编码规则做加减并做热量校验。样本不足 3 天会降级为基础目标并标注。想过放纵日说一句"今天放纵"，它会确认后写进 `cheat_days`，当日不参与任何计算。临时改训练安排（比如"改成不练了"），说一声它会重新定调覆盖今日目标。

**任务二：记录（CreateMealDraft）**。吃了什么直接告诉它，文字、图片、包装照片都行。数据查询优先级是：你的 `User_Vocab` 词典 > 模型知识库 > 联网（品牌官网 > 连锁官方营养表 > 权威数据库）。首次记录前会问一次"今天记过吗？"，若记过会先读今日已写入的行作为基线，之后默认简洁输出——只显示草稿表格和剩余配额，不重复展示基线明细，除非你说"展开基线"。图片识别或重量估算不确定的条目会标"待核实"，不会装作确定。剩余热量或某宏量低于当日目标 20% 且未到 22:00 会⚠️提醒，已超标出🚨。

**任务三：写入（AppendFoodRows）**。必须你明确说"写入""保存""提交""记录到表格"才真正写，模糊表达不算。草稿含"待核实"会再确认一次。走的是幂等写入：先查 `append_log` 最近 20 行有没有相同 `request_id`，没有才写主表，再写日志。主表写失败保留草稿让你重试，不会假装成功；主表成功但日志失败会如实告知。

**任务四：反馈（DailyNutritionFeedback）**。明确说"分析今天""反馈一下"才触发，平时不会主动输出。它先读日历拿到 `event_id`（会写在反馈末尾方便你关联），再综合当日摄入、餐次时间分布、估算风险给一段结构化反馈。"主问题"优先级在训练日和休息日不同：训练日看蛋白 > 碳水 > 热量 > 脂肪 > 进食时机，休息日看蛋白 > 脂肪 > 热量 > 碳水 > 进食时机。

## 设计上几条硬原则

营养数据、品牌、重量、份量一律不编造，拿不准就标"待核实"。所有阈值、公式、步骤都是硬编码，不会因为"感觉合理"就自己调整。每个任务只做它触发范围内的事，不串任务自由发挥。涉及等待确认的节点（是否重新定调、是否计入放纵日、是否写入待核实条目）必须停下来等你回复。连接器读写失败自动重试 2 次，仍失败就报错终止，不糊弄。

## 常用触发词速查

定调类用"起床""起床了""早上好""干活"；放纵日用"今天放纵""旅行""今天不管""随便吃"；改定调用"改成""不练了""换成""取消训练"；记录饮食直接描述吃了什么或发图；写入表格用"写入""保存""提交""记录到表格"；反馈用"分析今天""反馈一下"；说"结束"会清掉 `today_target`。

## 一些使用上的小提醒

同一天先说放纵、后说起床是允许的，它会撤回放纵记录再进定调流程。食物命名会保留影响营养的差异（比如不同口味的寿司不合并），营销词不进 `food` 字段。session 缓存跟着对话自然清理，但 `today_target` 例外，必须你说"结束"或次日重新定调时才换掉。
