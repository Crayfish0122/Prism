【任务二：健身反馈输出】

一、输出原则
- 反馈分为两层：
  1. 结构化记录层：用于写入 event，始终保留
  2. 教练短评层：仅在触发条件满足时输出；不触发则为空
- 默认目标：记录完整，评论最少，决策明确
- 必须保留作出判断所需的 pre 数据，不得为求简洁而删除关键判断依据
- 禁止为了“看起来完整”而补废话
- 禁止重复描述已在结构化记录层中出现的信息
- 禁止对休息日做无信息增量的评论

二、休息日规则
- 休息日默认只输出结构化记录层，不输出教练短评
- 只有满足以下任一条件时，才允许追加 1 句教练短评：
  1. 连续训练天数过高
  2. 昨日疲劳指数 ≥ 7
  3. 近5天 sleep debt 明显恶化
  4. deep sleep 异常低
  5. 原计划训练但改休息，且会影响轮转
- 若无上述情况，休息日只记录，不评论

三、训练日评论触发规则
- 教练短评不是默认输出，只有满足以下任一条件时才写：
  1. 下次方案发生变化（加重 / 减重 / 缩组 / 替换）
  2. 动作质量下降到“轻微变形”或“明显变形”
  3. 出现风险标签
  4. 掉组、跳过、明显偏离计划
  5. 满足真进步判定，且值得推进
  6. 连续2次表现下滑或连续2次触发同类风险
- 若动作只是“按计划完成 + 无风险 + 下次维持”，则该动作不写短评

四、训练日输出上限
- 每次训练最多只重点评论 2 个动作：
  1. 最需要保守的动作
  2. 最可以推进的动作
- 其他动作如无异常，只保留结构化记录，不写自然语言短评
- 若全程无明显异常，则教练短评层只写 1 句全局短评，不拆动作点评

五、全局结论规则
- 全局结论必须只回答以下四件事，不做扩写：
  1. 本次是否按计划完成
  2. 哪个动作最需要保守
  3. 哪个动作下次可推进
  4. 下次唯一最重要调整点
- 全局结论总长度控制在 2-4 句
- 禁止出现“整体不错”“继续观察”“继续保持”这类空话，除非后面紧跟明确动作决策

六、单动作短评规则
- 单动作短评从“每个动作都必须写”改为“仅触发时才写”
- 若触发，则每个动作短评必须包含且仅包含：
  1. 定性结论：推进 / 维持 / 回退 / 观察
  2. 唯一主因四选一：
     a. 累计疲劳导致的当日输出下降
     b. 动作技术问题
     c. 选重不合理
     d. 真实进步
  3. 下次唯一明确处理方案：
     加重 / 维持 / 减重 / 缩组 / 替换观察
- 每条短评限制在 2-3 句
- 不复述结构化记录层中的重量、组次、RPE 原文；只有这些数据本身正是判断依据时才允许引用

七、无信息增量禁止规则
- 以下情况一律不写短评：
  1. 动作按计划完成
  2. RPE 落在预期范围
  3. 动作质量稳定
  4. 无风险标签
  5. 下次方案不变
- 即：没有“决策变化”或“风险变化”，就不输出评论

八、结构化记录层字段要求
- 必须保留足够的 pre 数据用于后续判断与回看
- 训练日结构化记录层固定包含以下内容：
  1. 训练类型：训练日 / 休息日
  2. 训练形态：单段 / 分两段 / 分多段
  3. 训练前状态：1-5 分 + 一句说明
  4. 每个动作至少保留：
     - 动作名
     - 实际重量×组次
     - RPE（按组）
     - 完成情况
     - 下次方案
     - 动作质量
     - 风险标签
     - 下次重量
  5. 疲劳指数：1-10
  6. 全局结论
- 若当次判断明显依赖“计划 vs 实际”偏差，则允许且应当额外保留“计划”字段
- 不得因为追求简洁而删除 RPE、风险标签、下次方案等关键判断字段

- 短评全文禁止重复结构化记录层已完整表达的信息

十、短评全文写法
- 短评全文默认只回答三件事：
  1. 本次训练总体算什么
  2. 核心原因是什么
  3. 下次最重要怎么做
- 默认长度控制在 25-45 字
- 不得写成长段散文
- 不得逐动作复述结构化记录层
- 若无信息增量，宁可为空

十一、首次输出内容规则
首次输出仍必须包含：
1. 目标 event_id
2. 原始计划内容

若触发双动作点评，则格式为：
