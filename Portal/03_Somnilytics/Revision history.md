# 变更履历 · Somnilytics

> 记录 Prompt 各版本的核心变更与优化方向。

---

## v0.1.2（文件拆分与缓存机制版）

**变更方向**：降低指令维护成本，减少每次触发的规则加载开销

### 变更内容

**将单文件拆分为 Prompt.txt + sleep_rules.md**：初版把所有指令、解析规则、栏目约束、禁止事项都挤在同一份文件里，改动任意小细则都要通读整份 Prompt 才能定位修改点。v0.1.2 按"流程控制 vs 细则规范"切开——`Prompt.txt` 只保留触发条件、执行流程、输出模板和强制原则；`sleep_rules.md` 承载字段解析、派生指标、栏目分工、异常处理等全部细则。两者冲突时以 `Prompt.txt` 为准，优先级硬编码在 Prompt 顶部。

**新增规则缓存机制**：

- 单 chat 内 `sleep_rules.md` 只在首次触发「起床了」时读取一次，后续补问、主观收集、最终输出阶段全程走缓存
- 明确四种重读触发条件：用户声明规则更新 / 上传新版 `sleep_rules.md` / 开新 chat / 执行结果需要核对规则
- 补问阶段禁止展示规则来源、引用或选项说明，避免细则污染轻量对话

**当日缓存隔离键统一**：改用"最新一天"解析出的日期字段作为隔离键，同日期内的追问、补写、重新分析都写入同一份缓存；日期变化时强制清空旧缓存建立新缓存，堵住跨天数据混杂的漏洞。

**栏目分工约束显式化**：将"评级 / 证据 / 对比 / 结论 / 今晚"五行各自职责单独成节写入 `sleep_rules.md` 第 16 节，明确每行必须承担独立功能、不得换词复述前文。对应新增全局「去重总规则」：同一判断全文最多出现 2 次（一次用于结论、一次用于依据），换词复述一律删除。

**前一天建议执行改为默认自动判断**：不再列入 5 项固定提问，改由系统根据今天客观结果自动判定为 已执行 / 部分执行 / 未执行 / 无法判断 / 无前日建议。只有当建议属于不可直接观测类（如"睡前少刷手机""控制咖啡因"）且确实影响结论时，才允许追加一句补问，避免日常打扰。

## 已知限制与后续优化方向

**限制**

- 时区与日期边界随 Google Calendar 本地设置走，跨时区使用可能出现日期漂移，需手动对齐
- 睡眠事件的写入完全依赖上游流程（手动或外部同步），Somnilytics 只读不写，不负责创建或修复格式异常的事件
- 最近 7 天有效事件不足时只能降级为"基线不足"提示，无法从更久的历史数据自动兜底
- 规则缓存生效范围是单个 chat，跨 chat 需要重新加载，无法跨会话复用
- 主问题分类强制 6 选 1，遇到复合问题时会被归到"最主要"一类，可能掩盖次要问题

**后续优化方向**

- 评估是否需要周 / 月级别的纵向对比（当前只做单日 vs 7 天基线）
- 探索"复合主问题"的表达方式，让分类结果更准确地反映多维波动
- 研究规则版本号机制，让助手能在缓存中感知规则版本，不一致时自动重读
- 考虑把睡眠事件写入标准化为独立的上游小流程，与 Somnilytics 解耦但可联动
- 评估引入"基线平滑"策略（去极值、剔除明显异常日），减少单日极端数据对基线的拖拽

## v0.1.1（输出结构收敛版）

**变更方向**：压缩输出体积，收紧判断语法，避免模型自由发挥

### 变更内容

**输出从散段格式改为固定五行**：v0.1.0 的输出是多段式的"今日结论 / 依据 / 今日主问题 / 相对 7 天偏离 / 前日执行反馈"，容易出现段落间换词重复、堆同义句凑字数的问题。v0.1.1 压缩为固定五行结构——评级 / 证据 / 对比 / 结论 / 今晚，每一行承担独立功能，禁止跨行语义复述。

**主问题分类语法收紧**：规定只能使用"更像""主要表现为"这类弱判断表达；禁止"不是 A 也不是 B 而是 C"这种强排除式句式；每次必须只选 1 个最主要类型，不允许多选或排列组合。

**结构字段收敛为枚举值**：规定"结构"字段只允许 6 个取值——正常 / 深睡偏低 / 深睡偏高 / REM 偏低 / 效率偏低 / 信息不足。并明确 6 个值的优先级判定顺序（关键字段缺失 > 效率偏低 > 深睡异常 > REM 偏低 > 正常），避免模型在边界样本上摇摆。

**对比归类同样枚举化**：对比行的归类字段只允许 正常波动 / 已脱离近期常态 / 明显好于近期常态 三选一，不允许自由描述。

**主观字段展示强制转可读描述**：最终输出时禁止直接显示裸分（如"恢复感 2"），必须转成"偏差（2）"这类可读描述 + 括号分数的形式；补问阶段保持轻量，用户输入数字即可，映射留在后台处理。

**证据行字数与结构限制**：证据行最多写 4 个短分句、用分号分隔；主观部分最多引用与判断最相关的 1–2 项，禁止把 5 项主观记录再抄一遍。

## v0.1.0（初始版本）

**发布时间**：2025 年（初版，单文件形态，指令与规则混合在同一个 Prompt 中）

### 功能覆盖

- 触发词「起床了」驱动完整分析流程
- 从 Google Calendar「灵魂返厂」日历按 `start_time` 倒序读取最近 7 条有效睡眠事件
- 事件字段解析：日期、入睡/起床时刻、inbed、core、deep、rem、actual、debt_delta、status
- 派生指标计算：awake、efficiency、deep_ratio、rem_ratio、core_ratio
- 5 项主观字段补问（恢复感、精神度、起床方式、夜醒、入睡状态），一次一问、等回答再问下一条
- 主问题 6 分类：短睡型 / 长躺低效型 / 结构波动型 / 节律漂移型 / 主客观背离型 / 修复型
- 异常处理：非睡眠事件跳过、同日重复只留排序最靠前一条、字段缺失标「缺失」、格式异常标「解析失败」
- 有效事件不足 7 条时降级为"基线不足"分析，而非直接失败

### 核心设计决策

**纵向分析原则**：只看用户自己最近 7 天的睡眠变化，不做横向比较（与他人、与教科书标准），不引入睡眠外变量（饮食、运动、心情）。以数据边界换判断可靠性，避免小样本下的强发散。

**最新一天为唯一主分析对象**：最近 7 天只作基线、不单独展示明细。防止输出变成"7 天总结"稀释当天判断的焦点。

**事件格式严格校验**：title 必须为`睡眠`、description 按固定三行结构，任何偏离都被判为解析失败而非尝试容错解析。以牺牲一点灵活性换数据源可控性。

**只读不写**：Somnilytics 全程不更新 Google Calendar，分析结果只通过对话反馈给用户。保证数据源单一可信，避免自回写污染基线。

**主客观融合判断**：结论必须由"客观数据 + 主观输入 + 7 天基线"三者共同支撑，任一缺失时只能给弱判断（更像 / 主要表现为），不允许仅凭客观或仅凭主观下强结论。
