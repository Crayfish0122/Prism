# Alpha Sentinel

个人健康决策系统。围绕个人健康数据,做垂直、可执行、能反馈的助手系统。

## 系统定位

Alpha Sentinel 不是泛健康大杂烩,而是一套以个人数据为核心、以决策反馈为目的的健康助手系统。系统由三大助手构成:健身助手、营养助手、睡眠助手。三个助手不做并行开发,走"先纵后横"的串行路径——先把单个助手做深,再做时间纵深,最后做三助手之间的横向联动。

## 系统架构

整条数据流分为四层:输入层、中间层、后端分析层、输出层。

### 输入层:ChatGPT + Claude

用户的日常口述、记录、对话通过 ChatGPT 作为入口进行输入。Claude 直接调用 Google Calendar API,将 ChatGPT 产出的结构化内容写入 Calendar。Calendar 在系统中扮演"每日快照"的角色,每一天的健身、营养、睡眠数据都以日历事件的形式沉淀下来,作为当日状态的原始记录。

### 中间层:Google Apps Script

中间层由 GAS 完成,负责数据在 Calendar、Google Sheets、Excel 之间的流转。具体链路是 Calendar → Google Sheets → Excel:GAS 定时从 Calendar 抓取快照事件,汇总进 Google Sheets 的结构化表格,再同步到本地 Excel 供后端读取。中间层的核心职责是字段流转和数据接口,保证上游输入和下游分析之间的数据一致性。

### 后端分析层:Python

Python 后端运行在本地,从 Excel 中读取汇总数据,执行数据处理、趋势识别、决策评估的逻辑。后端不直接和用户交互,只负责把原始数据加工成可反馈的结论。

### 输出层:Telegram Bot

分析结果通过 Telegram Bot 推送给用户,形式包括 Daily Report、Weekly Report,以及助手层对用户的直接反馈。Telegram 是整套系统唯一的用户输出终端。

## 三大助手

健身助手是系统的第一个助手,也是后续营养、睡眠助手的样板。已完成第一阶段前端并上线,当前处于中间层维护期,主要工作是收 bug。

营养助手是健身助手之后的下一条主线,第一阶段待启动。定位不是独立并行开发,而是健身中间层稳定后的下一步。

睡眠助手排在营养助手之后,当前未进入主开发面。

## 开发阶段

项目分为三个阶段,按顺序推进,不跳阶段。

第一阶段是短期垂直挖掘,目标是把单个助手在短期场景下做深、做实、做可用,聚焦单一垂直主题,不追求跨领域联动。当前健身助手已完成第一阶段,营养助手和睡眠助手的第一阶段尚未启动。

第二阶段是中长期垂直挖掘,把每个助手从短期判断扩展到中长期趋势、阶段变化、持续反馈,时间维度拉长但仍是单助手纵向深化。当前未进入。

第三阶段是横向联动,打通健身、营养、睡眠三助手,形成跨助手的联动分析和反馈。必须等三个助手的第一阶段基础都打稳之后才开始。当前未进入。

## 当前状态

项目处于第一阶段。当前主线是健身助手中间层维护与 bug 收口,同时准备启动营养助手第一阶段。睡眠助手、第二阶段、第三阶段均在排队中,不占用当前开发资源。
